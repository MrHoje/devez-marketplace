# gajae-code 동기화 루틴

> hoje-code 스킬을 gajae-code 최신 버전으로 유지하는 자동 루틴

## 개요

| 항목 | 내용 |
|------|------|
| 작업 | gajae-code 최신 릴리스 확인 → hoje-code 스킬 동기화 → PR 생성 |
| 기준 버전 | gajae-code v0.8.1 |
| 스킬 매핑 | `plugins/hoje-code/README.md` 참고 |
| 적응 규칙 | 동일 파일 참고 |

## 동기화 스크립트 (GitHub Actions)

`.github/workflows/sync-gajae.yml` — 매일 09:00 KST 자동 실행

수동 실행: GitHub → Actions → Sync gajae-code → Run workflow

## Hermes 루틴 등록 (선택)

로컬 Hermes 에이전트로도 동기화 가능:

```bash
hermes cron create "0 0 * * *" \
  "gajae-code sync routine:
   1. Check https://api.github.com/repos/Yeachan-Heo/gajae-code/releases/latest
   2. Read marketplace.json version
   3. If current >= latest, respond [SILENT]
   4. If newer: run scripts/sync-gajae.ts, commit, push, create PR
   5. Report to Telegram" \
  --name "gajae-code-sync" \
  --deliver telegram \
  --workdir "<repo-local-path>"
```

## 수동 실행

```bash
cd <repo>
bun run scripts/sync-gajae.ts              # 최신 버전
bun run scripts/sync-gajae.ts --version=v0.9.0  # 특정 버전
```

## 주의사항

- PR은 자동 생성되며 auto-merge 되지 않음 → 사람이 리뷰 후 머지
- 적응 규칙이 최신 gajae-code 변경사항을 완전히 커버하지 못할 수 있음
- 새 스킬이 추가되면 `scripts/sync-gajae.ts`의 `SKILL_MAP` 업데이트 필요
