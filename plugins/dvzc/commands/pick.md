description: (DevezCode 전용) 다른 세션의 대화/작업 내용을 읽어와 현재 세션이 숙지한다
argument-hint: <세션이름>에서 작업한 내용 <읽어/merge해/요약해 등>
allowed-tools: Bash, Read
---
사용자가 다른 세션에서 진행한 내용을 이 세션이 파악해서 이어서 처리하길 원합니다.

요청: $ARGUMENTS
- 이 안에 대상 "세션 이름"(예: "세션 2")과 사용자가 원하는 동작(읽어/merge해/요약해/이어서 해 등)이 들어 있습니다.

절차:
1. **대상 세션 이름 추출** — 요청 문장에서 대상 세션 이름을 찾습니다(공백 표기 차이는 자동 보정됨).
2. **대화 내용 가져오기** — Bash 로 실행:
   ```
   node "${CLAUDE_PLUGIN_ROOT}/scripts/devez-pick.js" --session "<대상 세션 이름>"
   ```
   출력의 `읽을 파일:` 경로를 확인합니다. 대화는 `요청 N` 단위(사용자 요청 + 그 응답)로 정리되며, 길면 **자동으로 최근 요청만** 담깁니다. 전체가 필요하면 `--last 0`, 특정 개수만 원하면 `--last 30` 처럼 지정합니다.
3. **숙지** — 그 파일을 Read 로 읽어 대상 세션이 무엇을 논의/결정/작업했는지 파악합니다. (파일이 크면 나눠 읽어도 됩니다.)
4. **임시 파일 삭제** — 다 읽었으면 그 md 는 1회용 임시 파일이므로 즉시 삭제합니다. Bash 로:
   ```
   rm -f "<읽은 파일 경로>"
   ```
   (PowerShell 환경이면 `Remove-Item -Force "<경로>"`. 삭제 실패는 치명적이지 않으니 무시하고 진행하세요.)
5. **동작 수행** — 파악한 내용을 바탕으로 사용자가 요청한 동작(읽기/merge/요약/이어서 작업 등)을 수행합니다. 내용을 먼저 사용자에게 간단히 요약해 확인시키면 좋습니다.

주의: 대상이 claude 세션이 아니거나 한 번도 실행되지 않았거나 child 세션이면 transcript 가 없어 실패할 수 있습니다. 그 경우 오류 메시지를 사용자에게 그대로 전달하세요.
