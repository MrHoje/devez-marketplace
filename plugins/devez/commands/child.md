사용자가 다음 작업을 새 "자식 세션"에 위임하려고 합니다.

위임할 작업: $ARGUMENTS

아래 절차를 그대로 따르세요. (실제 코드 작업은 여기서 하지 말고 자식 세션에 맡깁니다.)

1. **세션 이름 짓기** — 위임할 작업을 분석해 **15자 이내의 짧은 한글 이름**을 만듭니다(예: "로그인버그수정", "README작성", "테스트추가"). 부모 이름이나 "위임" 같은 접두사는 넣지 마세요. 작업 내용만 압축.
2. **배경지식 추리기** — 지금까지의 대화 맥락에서, 이 작업을 처음 보는 세션이 **혼자서 수행**하는 데 필요한 배경을 정리합니다.
3. **자립적 브리핑 작성** — 다음을 모두 포함한 작업 지시서를 작성합니다:
   - (a) 배경/맥락 요약 (무엇을, 왜)
   - (b) 정확히 해야 할 일 (구체적 단계)
   - (c) 관련 파일·경로·제약
   > 자식 세션은 **같은 프로젝트 폴더**의 새 세션이라 코드베이스와 CLAUDE.md 는 이미 접근 가능합니다. 프로젝트에 이미 있는 내용은 장황히 옮기지 말고 "이 작업에 필요한 맥락"에 집중하세요.
4. **브리핑을 임시 파일로 저장** — Write 도구로 스크래치패드에 저장합니다(예: `child-brief.md`).
5. **자식 세션 생성 요청** — Bash 로 아래를 실행합니다(이름과 브리핑 파일을 넘김):
   ```
   node "C:/source/devezCode/plugins/devez-relay/scripts/send-new.js" --name "<15자이내 이름>" --brief-file "<브리핑파일 절대경로>"
   ```
6. **보고** — 명령 출력(성공/실패)을 사용자에게 한 줄로 전합니다. 자식 세션 생성·브리핑 주입은 DevezCode 가 처리합니다.
