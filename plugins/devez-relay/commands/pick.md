사용자가 다른 세션에서 진행한 내용을 이 세션이 파악해서 이어서 처리하길 원합니다.

요청: $ARGUMENTS
- 이 안에 대상 "세션 이름"(예: "세션 2")과 사용자가 원하는 동작(읽어/merge해/요약해/이어서 해 등)이 들어 있습니다.

절차:
1. **대상 세션 이름 추출** — 요청 문장에서 대상 세션 이름을 찾습니다(공백 표기 차이는 자동 보정됨).
2. **대화 내용 가져오기** — Bash 로 실행:
   ```
   node "C:/source/devezCode/plugins/devez-relay/scripts/devez-pick.js" --session "<대상 세션 이름>"
   ```
   출력의 `읽을 파일:` 경로를 확인합니다. (대화가 길면 `--last 40` 처럼 최근 N턴만 가져올 수 있습니다.)
3. **숙지** — 그 파일을 Read 로 읽어 대상 세션이 무엇을 논의/결정/작업했는지 파악합니다. (파일이 크면 나눠 읽어도 됩니다.)
4. **동작 수행** — 파악한 내용을 바탕으로 사용자가 요청한 동작(읽기/merge/요약/이어서 작업 등)을 수행합니다. 내용을 먼저 사용자에게 간단히 요약해 확인시키면 좋습니다.

주의: 대상이 claude 세션이 아니거나 한 번도 실행되지 않았거나 child 세션이면 transcript 가 없어 실패할 수 있습니다. 그 경우 오류 메시지를 사용자에게 그대로 전달하세요.
